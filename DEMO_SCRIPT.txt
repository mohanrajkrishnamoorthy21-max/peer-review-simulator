[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the workflow shown in the architecture diagram.
First, a user submits a paper abstract, which goes to the Security Checkpoint node. This node scrubs personal identifiers like emails and names, checks for prompt injection keywords, and ensures the text meets length requirements.
If it fails, it routes directly to the Final Output via a Security Event, aborting the process.
If it passes, the abstract goes to the Structure Analyzer node, which checks for necessary components like introduction, methods, results, and conclusion.
From there, the Citation Verifier agent runs to validate citation formats and DOI links.
Next, the Novelty Evaluator agent reviews originality by searching literature databases.
Then, the Language Critic agent critiques readability, grammar, and tone.
All these inputs converge at the Synthesizer Chair agent, which drafts a meta-review and suggests a verdict.
This brings us to the Human Approval node, where the system pauses for a human editor to sign off or input custom overrides.
Finally, the Revision Assistant agent takes the finalized feedback to draft specific, actionable improvement steps, which are combined into the final PDF-ready report.
